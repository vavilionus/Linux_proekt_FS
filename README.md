# myfs — простая собственная файловая система для Linux 6.12.x

Проект: блочная ФС поверх существующего блочного устройства с двумя
копиями суперблока, проверкой целостности (CRC32), монтированием через VFS
и набором IOCTL для управления.

## Содержимое директории

| Файл              | Назначение                                                 |
|-------------------|-----------------------------------------------------------|
| `myfs.h`          | Общий заголовок (диск, IOCTL, internal-структуры)         |
| `myfs_main.c`     | Точка входа модуля, параметры, регистрация `file_system_type` |
| `myfs_super.c`    | `fill_super`, форматирование, проверка CRC, кэш inode'ов  |
| `myfs_inode.c`    | inode-операции, `lookup`, `readdir`                       |
| `myfs_file.c`     | `read_iter` / `write_iter`, обработчики IOCTL             |
| `Makefile`        | Сборка модуля и тестовой программы                        |
| `myfs_test.cpp`   | Userspace CLI: тест read/write + вызов IOCTL              |
| `README.md`       | Этот файл                                                 |
| `run_example.txt` | Логи робного запуска с тестом всех операций               |

## Расположение данных на диске

```
блок:   0                 sb2_offset                  data_start
        │                       │                          │
        ▼                       ▼                          ▼
        ┌─────────────┬────────┬─────────────┬────────────┬──────────┬────┐
        │  SB copy 1  │  ...   │  SB copy 2  │  (unused)  │ file_0   │ ...│
        │  (1 блок)   │        │  (1 блок)   │            │ M блоков │    │
        └─────────────┴────────┴─────────────┴────────────┴──────────┴────┘
```

- Размер блока («сектора» в терминах задания) — **4096 байт**.
- `sb1_offset`, `sb2_offset` задаются параметрами модуля.
- `data_start = max(sb1_offset, sb2_offset) + 1`.
- `num_files = (total_blocks − data_start) / M`.
- Файлы называются `file_0`, `file_1`, … `file_{N-1}`.
- Каждый файл занимает ровно `M` подряд идущих блоков, размер фиксирован.
- В суперблоке хранится CRC32, при монтировании проверяется.
  Если копия #1 повреждена — восстанавливаем из копии #2 и наоборот.
  Если обе невалидны — диск форматируется с нуля.

## Параметры модуля

| Параметр             | По умолчанию | Описание                                  |
|----------------------|--------------|------------------------------------------|
| `device`             | (не задан)   | Имя блочного устройства (информационно)  |
| `sb1_offset`         | 0            | Блок первой копии суперблока             |
| `sb2_offset`         | 1024         | Блок второй копии суперблока             |
| `max_filename_len`   | 32           | Максимальная длина имени файла           |
| `max_file_blocks`    | 8            | Размер файла в блоках (M)                |

## Сборка

```bash
make            # модуль и userspace-программа
make module     # только myfs.ko
make test       # только myfs_test
make clean
```

Требуется установленный пакет с заголовками ядра (для 6.12.x):
`/lib/modules/$(uname -r)/build` должен существовать.

## Подготовка тестового устройства (loop-device)

```bash
# создаём файл-образ на 64 МБ
dd if=/dev/zero of=/tmp/myfs.img bs=1M count=64

# подключаем как блочное устройство
sudo losetup -fP /tmp/myfs.img
sudo losetup -l          
```

## Загрузка модуля и монтирование

```bash
sudo insmod ./myfs.ko \
    device=/dev/loop0 \
    sb1_offset=0 \
    sb2_offset=1024 \
    max_filename_len=32 \
    max_file_blocks=8

sudo mkdir -p /mnt/myfs
sudo mount -t myfs /dev/loop0 /mnt/myfs

# проверяем
ls /mnt/myfs
dmesg | tail -20
```

При первом монтировании на пустом устройстве модуль автоматически
отформатирует его (запишет обе копии суперблока и подсчитает CRC).

## Использование userspace-программы

```bash
# 1) Обходим все файлы, в каждый пишем и читаем случайное число.
sudo ./myfs_test test /mnt/myfs

# 2) Получаем CRC32-хэши содержимого всех файлов.
sudo ./myfs_test hashes /mnt/myfs

# 3) Получаем маппинг секторов для файла file_5.
sudo ./myfs_test mapping /mnt/myfs file_5

# 4) Обнулить все файлы.
sudo ./myfs_test zero /mnt/myfs
sudo ./myfs_test hashes /mnt/myfs 

# 5) Стереть ФС (зануляются обе копии суперблока).
sudo ./myfs_test erase /mnt/myfs
sudo umount /mnt/myfs
# при следующем монтировании ФС будет переформатирована
```

## IOCTL

| Команда                | Код                       | Действие                            |
|------------------------|---------------------------|------------------------------------|
| `MYFS_IOC_ZERO_ALL`    | `_IO('M', 1)`             | Обнулить все блоки данных           |
| `MYFS_IOC_ERASE_FS`    | `_IO('M', 2)`             | Стереть оба суперблока              |
| `MYFS_IOC_GET_HASHES`  | `_IOWR('M', 3, hashes)`   | Вернуть CRC32 каждого файла         |
| `MYFS_IOC_GET_MAPPING` | `_IOWR('M', 4, mapping)`  | Вернуть диапазон блоков файла       |

IOCTL принимаются как на корневом каталоге, так и на любом файле ФС.

## Размонтирование и выгрузка модуля

```bash
sudo umount /mnt/myfs
sudo rmmod myfs
sudo losetup -d /dev/loop0
```

## Лог запуска

Лог запуска находится в файле:

run_example.txt

В логе показаны:
- успешная сборка модуля;
- создание loop-device;
- загрузка модуля и монтирование ФС;
- отображение файлов в /mnt/myfs;
- успешный тест чтения/записи;
- работа IOCTL zero, erase, hashes, mapping;
- восстановление из резервной копии суперблока после повреждения первой копии.

## Особенности реализации

- Регистрация в VFS через современный API `init_fs_context` + `get_tree_bdev`
  (6.12.x).
- Чтение/запись — через `sb_bread` / `sb_getblk` (стандартный buffer-cache),
  с `mark_buffer_dirty` + `sync_dirty_buffer` после изменений.
- Размер inode'а расширен через свой кэш (`alloc_inode_sb`), что даёт
  возможность хранить `file_index`, `start_block`, `num_blocks` без
  дополнительных аллокаций.
- Все запрещённые задачей операции (rename/unlink/setattr) просто
  не объявлены — VFS вернёт `-EPERM/-EROFS`.
